01 Prove : Assignment - Experiment Shell & Hardcoded Classifier

Objective

Create an experiment shell that can be used to load data, apply a classification algorithm, and report results. As a placeholder, implement a classifier, "HardCodedClassifier", that always predicts the same class.

More than anything, the purpose of this assignment is to make sure you have the pieces in place to be successful in future assignments. Next week, we will program an actual classifier that will make use of the framework you put in place here.

Instructions

Write a Python program that can:

  1. Load a dataset containing many instances each with a set of attributes and a target value.

  2. Please use the popular Iris dataset, found at: http://archive.ics.uci.edu/ml/datasets/Iris . It contains 150 instances of three classes of Irises (50 per class), and four numeric attributes (in addition to the class name).

    Please note that this popular dataset is natively included in the Python scikit-learn package. You can easily obtain it with the following code:

    from sklearn import datasets iris = datasets.load_iris() # Show the data (the attributes of each instance) print(iris.data) # Show the target values (in numeric format) of each instance print(iris.target) # Show the actual target names that correspond to each number print(iris.target_names)

    In future weeks, you will need to be able to load data sets from text files as well, but for this assignment, loading from this datasets package is sufficient.

  3. Randomize the order of the instances in the dataset. Don't forget that you need to keep the targets matched up with the appropriate instance.
  4. Split the dataset into two sets: a training set (70%) and a testing set (30%).
  5. Create a class for a "HardCoded" classifier with the following methods:
    • train: accepts a training dataset (the instances and their corresponding targets). This method does not need to do anything for the HardCoded classifier.
    • predict: accepts an array of instances without targets, and for each one, predicts a class, and returns an array of predictions. For the "HardCoded" classifier the prediction can always be the same class (e.g., 0 or "setosa", but it doesn't really matter which one you choose).
  6. Instantiate your new classifier, "train" it with your training data, then use it to make predictions on the test data.
  7. Determine the accuracy of your classifier's predictions and report the result as a percentage.
  8. Create a new public repository at GitHub, and push your code to it.

Requirements

As always, you are encouraged to go above and beyond and take initiative in your learning. As described in the syllabus, meeting the minimum standard requirements can qualify you for 93%, but going above and beyond is necessary to get 100%. The following are the expectations for the minimum standard, with a few suggestions for going above and beyond.

Minimum Standard Requirements

  • The Iris dataset can be loaded
  • The order of the instances can be randomized
  • The dataset can be split into a training / test set
  • HardCoded classifier class is created with train and predict methods
    • Train accepts training data
    • Predict makes a prediction for the data it receives (at this point, it always predicts the same answer)
  • HardCoded classifier can be "trained" and "tested" on the Iris dataset and accuracy on the training set is reported.

Opportunities to go above and beyond:

  • In addition to loading data from the scikit-learn package, also, load data from CSV/text files
  • Handling non-numeric data types in CSV/text files
  • Adding user interface components (command line is just fine) to specify various parameters of an experiment (e.g., what dataset to use, what algorithm to use, how to split the training/testing data)
  • Adding n-fold cross validation
  • Any other ideas you have
