01 Prepare : Getting Started with Python

This semester we will be using Python for our coding exercises. Python is becoming increasingly popular in both industry and academia and is well-suited for data analysis.

This is not a "coding" course, and at this point in your program it is assumed that you can come up to speed on a new programming language on your own. But here are a few suggestions that may be beneficial.

  • CodeAcademy has a straightforward primer that walks you through the basics of the Python language.
  • It is recommended to use Python 2.7 (as opposed to the newer 3.4) because it is the version our textbook uses for its examples.
  • We will use NumPy, SciPy, matplotlib, and scikit-learn this semester. You can obtain these separately as needed, or you can get a distribution of Python that comes bundled with these and many more science/engineering libraries, such as: Anaconda ( http://continuum.io/downloads )
  • You can use whatever IDE you prefer (or none at all, see: https://xkcd.com/378/ ), but one good one you might consider is PyCharm ( https://www.jetbrains.com/pycharm/download/ ). The free community edition is great for our needs.

Good Programming Practice

While it will not be strictly enforced, as in a programming course, you are encouraged to use good style, code conventions, and programming practice.

The Google Python Style Guide is a good guide to base your code on. (Be aware that the textbook author does not adhere to these principles).

Finally, as a matter of good practice, be aware that if you put statements directly in your python files, it causes side effects when your code is imported into other files because that code is run. To avoid this, it is recommended that you structure your files with a main method as follows. This will ensure that your main method is run if your file is invoked directly and not if it is included in another module.

import sys

def main(argv):
    print "Hello World"

if __name__ == "__main__":
    main(sys.argv)
